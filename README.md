# Bom-Check

产品简介 (Overview)
在硬件产品从定型到量产的生命周期中，BOM（物料清单）的迭代核对是一项极度消耗精力且极易出错的流程。
本工具是一个基于 Python 构建的轻量级、全自动 BOM 差异分析引擎。它能够智能读取新旧两版 BOM 文件，瞬间完成底层交差集逻辑运算，并输出一份排版严密、自动对齐的「二维矩阵式 Excel 差异报告」。它旨在彻底消除人工比对 PCBA 样板数据时的“上下文断层”，实现降维打击式的效率提升。

release 1.0

智能协议嗅探与优先级仲裁 (Multi-Format Arbiter)
---当前版本允许导出四种格式文件bom：.xlsx .xlsm .xls .csv。
---支持解析格式优先级顺序：.xlsx > .xls > .xlsm > .csv。

文档名字必须遵循：new_bom.x/old_bom.x

防呆与防御性编程 (Defensive Logic)
---如果新旧 BOM 的列模板不一致（如新版漏导了 Footprint 列），程序不会崩溃，而是优雅地在单元格内标记 [新版缺失此列]。
---自带阻塞挂起机制，防止 EXE 运行完毕后终端界面闪退。

注：支持 .xlsx / .xls / .csv 后缀。如果使用 .csv，请确保已手动删除文件最顶部的无用项目说明文字，让表头（如 Designator）保持在绝对的第一行。
